New Features

Email Viewer UI

  • Added a (?) shortcut icon to the "Info Overlay" in the top bar.
  • Info Overlay now has a close button in the upper right hand corner.
  • You can now shrink (or less likely, expand) your desktop email preview to fit to the screen. Helpful to get a quick look at your entire email.
  • Info Overlay has been updated to show more of the shortcuts available to you.
  • After refreshing an email preview, Korra now remembers if you had guidelines turned on, hidden content visible, you were in fullscreen mode (hiding the left pane and the mobile view), or if you toggled dark mode.

Email QA

  • A large block of "spam" words that we used to highlight have now been removed. Further research has proven that inbox placement has very little to do with supposed "spam" words so pointing them out in the email preview is unecessary.
  • A new link validation option alerts you to duplicate links in your email. When enabled, if an exact copy of a link is seen a second time or more you will see an error message. Enable this if you would like to be reminded to differentiate your links to improve the accuracy of your reporting. This option is off by default on fresh installations.
  • The accessibility checker will no longer warn you to add a lang="" attribute on some elements that do not need it.

Email Gallery

A new feature that will display recent emails in a collage style view. Access it from the top bar when viewing any email. This is still a work in progress.

Other

  • Created a "What's New" page that pulls in the changelog for the extension. It opens when the extension is updated and also appears in the "Info Overlay" when viewing an email.

Bug Fixes

  • The ability to toggle link markers on and off wasn't working because of some renaming of elements.

Maintenance

  • Added back in permissions that allowed us to access any page on the internet. It was previously removed because Google has been cracking down on extensions requesting more permissions than they need. After attempting to tighten up our manifest it became clear that we would need those broader permissions.
  • Moved all page zoom related code to it's own .js file for better organization.
  • The tools pane now uses sessionStorage instead of querystring parameters to remember a users setting across refreshes.
