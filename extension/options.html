Link Validation
Korra can check your links for you automatically against a list of default formatting rules and custom rules you create. Korra can also follow your links to check that they are loading properly and give you feedback on how to fix them.

General

Automatically validate all links in an email as soon as its loaded. You can manually check or re-check your links without reloading on a per-email basis from within the email you're previewing.
By default Korra will check links even if they have a rel attribute of 'nofollow'. Uncheck this if you'd like to use rel="nofollow" as a way to have Korra ignore certain links.
If you don't want to revalidate good links over and over again, enable this setting to force Korra to only validate new links or links that failed to pass all validation rules on a previous check. Links are cached on a per-email basis. Identical links in separate emails will both be checked even if one is already cached.

You can recheck cached links at any time by clearing the cache of all links related to the email you're currently previewing from within the email preview UI. You can also clear Korra's entire cache of valid links with the button below.
Prevent Korra from checking a link by adding it to the disallow list. Enter each URL on its own line.
(Example: www.mysite.com will stop these links from being validated: www.mysite.com/index.html or www.mysite.com/path/index.html)

Format Validation

These settings focus on validating the formatting of your links.

When enabled if there is an anchor tag without an href attribute you will see an error message. If disabled, Korra will skip these links entirely and provide no feedback.
When enabled, if the link url ends with a '#' you will be warned.
When enabled, if the link url is empty (href="") you will be warned.
Some ESP tags include characters that would normally be invalid in a link (e.g. *|tag|* or %%variable%%). Enable this to prevent Korra from potentially marking your links as invalid when it finds one.
Note: Provide the formatting that your ESP uses in order for this to work. Enter this in the "ESP Options" section.

Loading Validation

Korra can follow your links to verify that they load properly.

When enabled Korra will follow your links to verify that they load properly. If the link fails to load for some reason you'll receive an error message with the reason to help you diagnose the problem.
HEAD is quicker, but sometimes doesn't reflect the true HTTP status of a page, use GET if you're getting weird results. GET is required to be able to parse the document for an element corresponding to what is after the hashtag by ID.
Hashtags can link to elements on the destination page by their id attribute. For example, <a href="page2.html#elementID">Link</a> would link to an scroll to an element with id="elementID" when the page has loaded. Enabling this will allow Korra to automatically check that all links with hastags have a matching element. When enabled, if the page loads but the corresponding element is not found, the link will be marked with a warning.
Request Type: GET is required
